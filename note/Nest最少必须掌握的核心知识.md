# Nest 最少必须掌握的核心知识

这份清单的目标不是“把所有知识点都背下来”，而是帮你分清：

- 哪些内容已经够你写基础接口
- 哪些内容必须补，不然项目一复杂就会卡住
- 学 Nest 时应该先学什么、后学什么

## 1. 现在这批装饰器，够不够用？

如果你的目标是：

- 看懂基础教程
- 写简单 CRUD 接口
- 接收参数并返回数据
- 理解模块、控制器、服务之间的关系

那么你现在掌握的这些装饰器，**已经够用了**。

可以说，它们足够支撑你完成：

- 模块拆分
- 控制器编写
- Service 注入
- 路由定义
- 参数提取
- 基本响应控制

也就是说，**入门阶段没有问题**。

但如果目标变成下面这些：

- 写中小型项目
- 做登录鉴权
- 做参数校验
- 统一异常处理
- 封装公共能力
- 看懂公司里的 Nest 项目

那只会这些装饰器就不够了。

## 2. 真正必须补的核心知识

下面这些内容，才是 Nest 从“会写接口”到“能做项目”的关键。

### 2.1 模块化与依赖注入

这是 Nest 最核心的一层，比单个装饰器更重要。

你至少要搞懂：

- `Module` 的作用
- `imports`、`providers`、`controllers`、`exports` 分别是什么
- provider 为什么能被注入
- 为什么有的 service 能注入，有的不能注入
- 跨模块复用 service 时为什么要 `exports`

如果这块不清楚，项目一拆模块就会乱。

### 2.2 DTO + Pipe + ValidationPipe

只会 `@Body()` 不够，真正项目里更重要的是：

- 如何定义 DTO
- 如何校验参数
- 如何把字符串转换成需要的类型
- 如何在入口统一拦截非法参数

这是你写“稳定接口”的基础。

常见组合：

```ts
@Post()
create(@Body() dto: CreateUserDto) {
  return dto
}
```

配合全局 `ValidationPipe` 后，Nest 才真正进入“工程化”的状态。

### 2.3 Guard

Guard 是登录校验、权限控制最常见的一层。

你至少要知道：

- 它是干什么的
- 它和中间件有什么区别
- 它什么时候执行
- 它怎么配合 metadata 使用

如果项目里有：

- 登录态校验
- token 校验
- 角色权限控制

那基本一定会用到 `@UseGuards()`。

### 2.4 Interceptor

Interceptor 常用于：

- 统一包装返回结果
- 记录日志
- 统计耗时
- 响应数据转换

它是 Nest 很有代表性的能力，很多横切逻辑都会放到这里。

### 2.5 Filter

Filter 主要负责统一异常处理。

你至少要理解：

- 为什么业务里要统一异常格式
- `HttpException` 怎么处理
- 自定义异常过滤器是怎么接管错误的

你现在项目里已经有过滤器文件了，这块可以顺着代码继续学。

### 2.6 自定义装饰器

项目里常见的不是只用官方装饰器，而是自己封装：

- `@User()`
- `@Roles()`
- `@Public()`

这类装饰器通常和 `SetMetadata`、Guard 配合使用。

学会它之后，代码会更整洁，也更贴近真实项目写法。

## 3. 最值得优先掌握的 20%

如果只抓最重要的部分，优先学这几块就够了：

1. `Module`、`Controller`、`Service` 的关系
2. 依赖注入是怎么工作的
3. `@Param()`、`@Query()`、`@Body()` 的用法
4. DTO 和 `ValidationPipe`
5. Guard 的基本用法
6. Interceptor 和 Filter 的基本职责
7. 全局配置的入口在哪里

把这 7 件事学明白，已经能覆盖大多数基础项目场景。

## 4. 推荐学习顺序

建议按下面顺序学，不容易乱：

### 第一阶段：先能写接口

- `@Module()`
- `@Controller()`
- `@Injectable()`
- `@Get()`、`@Post()`、`@Put()`、`@Delete()`
- `@Param()`、`@Query()`、`@Body()`

目标：能写一个完整 CRUD。

### 第二阶段：让接口更像真正项目

- DTO
- `ValidationPipe`
- 全局 Pipe
- 异常处理
- `@HttpCode()`、`@Header()`

目标：让接口有校验、有错误处理、有统一输出思维。

### 第三阶段：进入工程化

- Guard
- Interceptor
- Filter
- `SetMetadata`
- 自定义装饰器

目标：能做登录、权限、日志、统一响应等项目能力。

### 第四阶段：理解 Nest 设计思想

- 动态模块
- 自定义 provider
- 生命周期钩子
- 中间件、Guard、Interceptor、Pipe、Filter 的执行顺序

目标：从“会用”变成“看得懂框架结构”。

## 5. 一句话结论

你的这批装饰器：

- **够入门**
- **够写基础接口**
- **不够支撑完整项目**

接下来最应该补的不是继续背更多装饰器，而是补这 4 个核心能力：

- 模块化和依赖注入
- DTO 与参数校验
- Guard
- Interceptor / Filter 的职责和配合

## 6. 建议你下一步学什么

如果你想继续往下学，推荐顺序是：

1. 先把 `Module`、`Service`、依赖注入彻底搞懂
2. 再学 DTO + `ValidationPipe`
3. 再学 Guard
4. 最后学 Interceptor 和 Filter

这个顺序最贴近真实开发，也最不容易学乱。
